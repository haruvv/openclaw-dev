# Claude Code 作業指針

このリポジトリは openclaw-dev 開発エージェントが作業する対象リポジトリ。
GitHub Actions 上で Claude Code が自動的に Issue を実装し PR を作成する。

## 作業フロー

1. Issue の内容を読んで要件を把握する
2. 必要なファイルを調査・読み込む
3. 実装する（テストファーストを推奨）
4. CI が通ることを確認する
5. PR を作成して Issue をリンクする（`Closes #N` を PR 本文に含める）

## コーディング規約

- 既存コードのスタイルに従う
- 型安全を優先する（TypeScript を使う場合は `any` を避ける）
- テストは実装と同時に書く
- コメントは自明でないロジックにのみ追加する

## PR 作成ルール

- ブランチ名: `issue-{N}-{kebab-case-description}`
- PR タイトル: Issue タイトルをそのまま使う
- PR 本文に `Closes #N` を含める
- スクリーンショットや動作確認結果があれば添付する
